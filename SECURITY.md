# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |

Only the latest major version is currently supported with security updates.

## Reporting a Vulnerability

Please report security vulnerabilities using GitHub's private vulnerability reporting feature:

**[Report a vulnerability](https://github.com/amagyar/ccl/security/advisories/new)**

Do not create a public issue for security vulnerabilities.

### What to Include

When reporting a security vulnerability, please include:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested mitigations (if available)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 5 business days
- **Resolution Timeline**: Depends on severity; critical issues will be prioritized

### Responsible Disclosure

We ask that you:

1. Not disclose the vulnerability publicly until a fix is released
2. Provide reasonable time for us to investigate and fix the issue
3. Not access or modify data that doesn't belong to you

Thank you for helping keep ccl secure!
